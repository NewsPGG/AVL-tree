# Отчёт по анализу производительности: Списки vs АВЛ-деревья

### 1. Таблица результатов
Данные получены с помощью утилиты `perf stat`. Замеры проводились на базе из 10 000 записей.

| Сценарий | Список (List)   | АВЛ-дерево (AVL) | Разница  |
| :--- |:----------------|:-----------------|:---------|
| **№1 (50 000 find)** | 0.72s           | 0.01s            | **~72x** |
| **№2 (10 000 insert + find)** | 0.28s | 0.01s  | **~28x** |

---

### 2. Анализ профиля (perf report) для связного списка
Профилирование "горячих" функций с помощью `perf record` выявило следующие точки нагрузки:

1. **`find` (Overhead ~94%):** Основная функция, потребляющая ресурсы. Высокий процент вызван необходимостью линейного итерирования по списку до нахождения нужного кода.
2. **`strcmp` (Overhead ~80%):** Внутри функции `find` больше всего времени уходит на сравнение строк. Это ожидаемо, так как количество сравнений растет пропорционально $O(n)$.
3. **`node->next` (Cache Stalls):** В режиме `perf annotate` заметно значительное время ожидания данных из памяти. Процессор "простаивает" при переходе по указателям, так как узлы списка разбросаны в куче (Heap), что вызывает высокие показатели `cache-misses`.

---

### 3. Объяснение разницы
Превосходство АВЛ-дерева объясняется логарифмической природой алгоритма.

**Асимптотика:**
* **Список:** Поиск — $O(n)$, Вставка — $O(n)$ (с учетом поиска позиции для сохранения порядка).
* **AVL:** Поиск — $O(\log n)$, Вставка — $O(\log n)$.

В то время как списку для 10 000 элементов требуется в среднем **5 000 сравнений**, АВЛ-дерево ограничивается максимум **14 сравнениями** (высота дерева). При вставке (`insert`) дерево также выигрывает: несмотря на необходимость балансировки (поворотов), поиск места для нового узла в списке всё равно остается линейно зависимым от его размера.

---

### 4. Вывод для директора
Переход на АВЛ-деревья позволит на порядки сократить время отклика системы регистрации. Текущая архитектура обеспечивает мгновенную обработку данных и обладает высоким запасом масштабируемости, исключая риск возникновения очередей и зависаний системы при росте базы аэропортов.
